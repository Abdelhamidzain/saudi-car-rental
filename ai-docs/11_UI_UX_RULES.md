# 11 — UI/UX Rules

## Design Direction

The platform should feel:
- Trustworthy.
- Clean.
- Saudi-market appropriate.
- Fast.
- Professional.
- Useful, not overly flashy.

## Customer Experience

Main flow:
1. Search.
2. Compare.
3. Understand price is starts-from.
4. Submit request.
5. Rental company contacts customer.

## Important UI Principles

- Keep lead form simple.
- Show price disclaimers clearly.
- Show positive badges only.
- Make compare table easy to scan.
- Make phone CTA clear.
- Avoid confusing booking language.

## Offer Card Must Show

- Company name.
- Car name/model.
- City/branch.
- Starts-from daily price.
- Monthly price if available.
- Last price update.
- Insurance included or not.
- Mileage limit if available.
- Airport delivery if available.
- CTA: Submit request / Request this offer.

## Do Not Show

- Medium trust.
- Low trust.
- Negative company labels.
- Final price unless confirmed.
- Public internal score.

## Admin UX

Admin dashboard must reduce manual work.

Admin should have:
- Copy WhatsApp message.
- Open WhatsApp.
- Quick status update.
- Quick follow-up buttons.
- Clear timeline.
- Duplicate lead warning.
- Follow-up reminders.

## Company Dashboard UX Future

Company dashboard should be simple:
- New leads.
- Contact customer.
- Mark contacted.
- Mark won/lost.
- Update offers.
- Update prices.
- View performance.
